算法综合作业一实现了Dijkstra's shortest path algorithm，其中有几点优化，只是为了说明算法没有采用动态分配内存，实际中可以根据需要动态地分配内存大小。

算法假设了输入的节点编号是变化范围不大的，如果存在下述情况（1,99999999）只有两个节点，则空间效率非常低，需要优化处理。

优化1.因为是稀疏图所以采用邻接矩阵，节省了存储空间。
优化2.用一个最小堆保存当前最小定点集合，可在O(lgE’)的时间内完成各种单的操作，用数组映射heappos[]映射节点在堆中的位置，可以将Update（）操作优化为O(lgE’)。所有操作的总时间复杂度上限为O(E’lgE’)。
优化3.将每个更新的节点都加以保存（PUSH1与PUSH2），使每次运行完毕后对所操作的节点进行返回修改，避免了O(E)的初始化复杂度。
优化4. 一次分配好算法所需存储总边数的空间edges[EDGMAXSUM<<1]，由算法保证各个边的存储，省略了为各个边动态分配内存的耗时。
优化5.搜到结束节点即完成算法，避免冗余计算。
